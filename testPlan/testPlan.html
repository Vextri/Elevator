Test Plan for the VI Elevator Project

Test Objectives and Scope

The purpose of testing the Project's VI elevator is to confirm CAN, Ethernet, and Wireless communication with the Raspberry Pi. The testing will verify that all components can successfully communicate with each other and exchange data as expected.

Test Types

Communications Testing and Verification

  • CANBus
  • Wireless

Test Procedures

CANBus

Transmission of CAN Messages

Objective: Verify that the supervisory controller can transmit CAN messages to itself, MCUs, and Arduino.
Procedure: 
Program the supervisory controller to send messages on the CAN bus.
Verify that messages are received by the intended recipients (itself, MCUs, Arduino).
Ensure that payloads are correctly formatted and as expected.

Listening for CAN Messages (Message Filtering)

Objective: Ensure that the supervisory controller correctly listens for messages and filters out unnecessary traffic.
Procedure: 
Set up a specific MCU to periodically broadcast messages on the CAN bus.
Verify that only the supervisory controller is receiving the messages (filters are working correctly).
Monitor and log message activity to verify the CAN bus is correctly receiving and processing.

Message Payload Verification

Objective: Verify the integrity and correctness of payloads within the messages.
Procedure: 
The supervisory controller should read and decode the payloads from messages on the CAN bus.
Check for correctness in the payload data for different types of messages (move floors, control signals, etc.).

Wireless

Wireless Communication Setup

Objective: Ensure proper setup of the wireless communication (Wi-Fi, Bluetooth, etc.) between the supervisory controller and external devices.
Procedure: 
Set up the wireless module on both the supervisory controller and any external devices (e.g., Arduino, MCU).
Verify successful connection between devices via the wireless network.

Wireless Communication Stability

Objective: Verify the reliability of wireless communication.
Procedure: 
Test message transmission under various conditions (e.g., interference, distance).
Measure latency, packet loss, and data throughput over the wireless link.

Website Controller Integration

Objective: Test communication between the supervisory controller and the website interface.
Procedure: 
Use the website to control supervisory controller features.
Monitor Ethernet communication for performance and correctness.
Verify that the website can send and receive valid commands.
